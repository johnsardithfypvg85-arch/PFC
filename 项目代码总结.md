# 项目代码总结

## 1. 项目概述

本项目是一个基于 TI C2000 `F28034` 平台实现的单相 PFC 闭环控制工程，工程名为 `Demo-PFC-CloseLoop`。  
从代码结构和控制流程来看，它不是简单的外设例程，而是一套较完整的数字电源控制程序，包含：

- 系统时钟与外设初始化
- PWM 触发 ADC 采样
- 输入电压同步与锁相
- 电压环、电流环双闭环控制
- CCM/DCM 前馈补偿
- 软启动与状态机管理
- 过压、欠压、过流保护
- 与后级 DCDC 的 SCI 通信

主程序入口位于 [main.c](/E:/TI/Demo-PFC-CloseLoop/main.c)，真正的控制任务主要在多个中断服务函数中完成。

## 2. 工程结构

项目核心源码主要由以下几个文件组成：

- [main.c](/E:/TI/Demo-PFC-CloseLoop/main.c)：系统初始化、外设配置和中断使能
- [InterruptADC.c](/E:/TI/Demo-PFC-CloseLoop/InterruptADC.c)：ADC 快速中断，执行电流快环与 PWM 更新
- [Interrupt1kHz.c](/E:/TI/Demo-PFC-CloseLoop/Interrupt1kHz.c)：1kHz 中断，执行电压慢环和 RMS 计算
- [Interrupt200Hz.c](/E:/TI/Demo-PFC-CloseLoop/Interrupt200Hz.c)：200Hz 中断，执行状态机、保护和通信调度
- [SCIcom.c](/E:/TI/Demo-PFC-CloseLoop/SCIcom.c)：SCI 通信协议处理
- [InterruptADC.h](/E:/TI/Demo-PFC-CloseLoop/InterruptADC.h)：全局控制变量、结构体和关键函数声明
- [Table.h](/E:/TI/Demo-PFC-CloseLoop/Table.h)：正弦表和开方查表

此外，`DSP2803x_common` 和 `DSP2803x_headers` 是 TI 提供的底层驱动与头文件。

## 3. 整体控制架构

本项目采用典型的“主函数初始化 + 多中断分层控制”架构。

### 3.1 主函数职责

[main.c](/E:/TI/Demo-PFC-CloseLoop/main.c) 主要完成：

- 关闭看门狗
- 初始化系统时钟和中断控制器
- 配置中断向量表
- 将关键控制函数从 Flash 拷贝到 RAM 运行
- 初始化 Timer、PWM、ADC、GPIO、SCI、Trip Zone
- 使能全局中断和各功能中断

主循环本身为空，说明系统属于典型的中断驱动式嵌入式控制程序。

### 3.2 三层控制节拍

项目按照不同实时性要求，将任务分配到三个主要中断中：

1. `ADC` 中断  
   跟随 PWM 采样触发，频率最高，主要负责电流快环。

2. `1kHz` 中断  
   负责电压环、有效值计算、参数更新和软启动。

3. `200Hz` 中断  
   负责状态机、输入电压保护和 SCI 通信。

这种结构符合数字电源控制的常见实现方式：快环负责电流，慢环负责电压，更慢层负责状态与通信。

## 4. 核心控制流程

### 4.1 ADC 快速中断

[InterruptADC.c](/E:/TI/Demo-PFC-CloseLoop/InterruptADC.c) 是项目最核心的控制模块。`ISR_ADC()` 中断执行流程如下：

1. `ADCSample()`  
   采样 MOS 电流、母线电压、输入电压 L/N。

2. `PhaseLockLoop()`  
   根据输入电压过零与计数实现简化锁相，得到正弦表位置。

3. `FeedCal()`  
   计算 CCM/DCM 前馈量，降低电流环调节负担。

4. `IrefCal()`  
   根据电压环输出、输入电压幅值和 RMS 信息生成电流参考。

5. `CurrentLoop()`  
   执行电流环 PI 运算。

6. `RegReflash()`  
   更新 `EPwm1Regs.CMPA`，输出新的占空比。

7. `Icalibrate()`  
   在 DCM 模式下修正电流采样。

8. `HwOcp()`  
   检查硬件过流封波并决定是否进入故障状态。

这一层决定了 PFC 电流波形跟踪质量，是系统最关键的实时闭环部分。

### 4.2 1kHz 慢速中断

[Interrupt1kHz.c](/E:/TI/Demo-PFC-CloseLoop/Interrupt1kHz.c) 负责慢速调节和参数更新，主要包括：

- `VbusAvgCal()`：对母线电压做滑动平均
- `VloopKpKiCal()`：根据状态调整电压环增益
- `VrefCal()`：生成母线电压参考并实现电压软启动
- `VoltageLoop()`：执行母线电压环 PI 调节
- `VbusLimit()`：母线过压限制
- `VbusUVP()`：母线欠压保护
- `VIacRmsCal()`：计算输入电压、电流有效值
- `DCMFeedCal()`：更新 DCM 前馈系数
- `IloopKpKiCal()`：根据工况调整电流环参数
- `Feedss()` / `Irefss()`：前馈和电流参考软启动

这一层的作用是给快环提供更合理的目标值和参数，使整个系统在动态和稳态下都保持较好的性能。

### 4.3 200Hz 状态机中断

[Interrupt200Hz.c](/E:/TI/Demo-PFC-CloseLoop/Interrupt200Hz.c) 负责更高层级的运行管理：

- `StateM()`：状态机调度
- `VacCheck()`：输入电压过压/欠压保护
- `SciCom()`：SCI 通信处理

状态机共分为 5 个状态：

- `Init`：初始化状态，完成变量和零偏校准
- `Wait`：等待状态，检测母线电容预充是否完成
- `Rise`：软启动状态，逐步建立 PFC 输出
- `Run`：正常运行状态
- `Err`：故障状态，关闭 PWM 并等待恢复

这个状态机把“上电、等待、启动、运行、故障”串成了完整的生命周期。

## 5. 数据结构设计

项目将控制相关的数据集中定义在 [InterruptADC.h](/E:/TI/Demo-PFC-CloseLoop/InterruptADC.h)，主要包括：

- `_ADI`：ADC 采样结果，如电流、电压、偏置
- `_PLL`：锁相环状态量
- `_VAC`：输入电压相关量，如峰值、RMS、构造正弦
- `_IAC`：输入电流相关量，如 RMS、参考值
- `_VBUS`：母线电压平均值和参考值
- `_FEED`：前馈控制量
- `_ILOOP`：电流环 PI 参数和积分量
- `_VLOOP`：电压环 PI 参数和积分量
- `_FLAG`：系统状态、故障和软启动标志

这种设计比较符合数字控制工程代码风格，优点是变量职责清晰，便于跟踪整个控制链条。

## 6. 外设配置特点

从 [main.c](/E:/TI/Demo-PFC-CloseLoop/main.c) 的初始化代码可以看出，项目对外设做了明确分工：

- `ePWM1`：PFC 主功率 PWM 输出，同时触发 ADC 采样
- `ePWM4`：模拟 DAC 调试输出
- `ADC`：同步采样电流、母线电压和输入电压
- `Timer0`：产生 1kHz 中断
- `Timer1`：产生 200Hz 中断
- `Trip Zone + COMP1`：硬件过流快速封波保护
- `SCI-A`：与后级 DCDC 进行串口通信
- GPIO：继电器控制、PFC OK 信号和调试引脚

尤其值得注意的是，快环函数被放到了 `ramfuncs` 中运行，这说明作者明确考虑了实时性和 Flash 执行延迟问题。

## 7. 通信功能说明

[SCIcom.c](/E:/TI/Demo-PFC-CloseLoop/SCIcom.c) 实现了一个比较轻量的串口通信协议，用于 PFC 和后级 DCDC 之间交换状态与测量值。

主要特点如下：

- 使用固定长度帧
- 首字节区分消息类型
- 末字节使用校验和
- 接收后按不同命令类型处理
- 必要时回传故障、状态、输入电压、电流信息

这个通信模块更偏向板级内部通信，而不是复杂的人机交互协议。

## 8. 项目优点

从代码实现角度看，这个项目有以下优点：

- 控制层次划分清晰，快环、慢环、状态机职责明确
- 双环控制、前馈、软启动、保护机制较完整
- 中断组织方式符合数字电源控制常规架构
- 关键快环函数放到 RAM 中，考虑了实时性
- 状态机逻辑完整，覆盖上电到故障恢复全过程
- 具备与后级 DCDC 的联动接口，工程化程度较高

## 9. 存在的问题与风险

虽然整体结构较完整，但从可维护性和健壮性角度看，也存在一些明显问题：

### 9.1 参数硬编码较多

大量控制参数、保护阈值、比例系数直接以宏的方式散落在源码中，调试和维护成本较高。  
后续若要适配不同功率段或不同输入输出规格，建议建立统一的参数配置层。

### 9.2 全局变量耦合较强

多个中断共享大量全局变量，虽然在裸机实时系统中比较常见，但后续修改时很容易引入时序问题。  
如果继续扩展功能，建议对关键共享变量的更新时机进行更明确的约束。

### 9.3 查表数据直接写在头文件中

[Table.h](/E:/TI/Demo-PFC-CloseLoop/Table.h) 中直接定义了数组，而不是采用 `extern` 声明配合 `.c` 文件定义。  
当前虽然可以工作，但如果被多个源文件同时包含，会带来重复定义风险。

### 9.4 注释较多但命名不完全统一

代码注释非常丰富，这对学习很有帮助；但部分变量和函数命名中英文混用、缩写较多，阅读时需要结合上下文理解。

## 10. 结论

总体来看，这个项目是一套较完整的数字 PFC 闭环控制工程，具备较强的学习和参考价值。  
其核心思想可以概括为：

- 使用 PWM 同步 ADC 采样实现电流快环
- 使用 1kHz 慢环实现母线电压稳定控制
- 使用 200Hz 状态机管理启动、运行、故障与通信
- 通过锁相、前馈和软启动改善输入电流质量与启动过程
- 通过过压、欠压、硬件过流保护提高系统安全性

如果后续要继续深入理解该项目，建议优先按以下顺序阅读源码：

1. [Interrupt200Hz.c](/E:/TI/Demo-PFC-CloseLoop/Interrupt200Hz.c)：先理解状态机和启动流程
2. [Interrupt1kHz.c](/E:/TI/Demo-PFC-CloseLoop/Interrupt1kHz.c)：再理解电压环和 RMS 计算
3. [InterruptADC.c](/E:/TI/Demo-PFC-CloseLoop/InterruptADC.c)：最后深入分析快环、前馈和 PWM 更新

这样最容易把整套控制逻辑串联起来。
