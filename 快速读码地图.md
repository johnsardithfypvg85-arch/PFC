# PFC 项目一页式读码地图

> 目标：帮助新同学在 30~90 分钟内快速看懂本项目的控制架构与关键代码路径。

## 1. 先看哪里（建议顺序）

1. `main.c`
   - 看中断向量绑定、外设初始化、全局中断使能。
2. `Interrupt200Hz.c`
   - 先理解状态机（`Init/Wait/Rise/Run/Err`）与故障恢复逻辑。
3. `Interrupt1kHz.c`
   - 理解电压慢环、RMS 计算、参数调度和软启动。
4. `InterruptADC.c`
   - 最后深入电流快环、PLL、前馈与 PWM 更新。

---

## 2. 调度总览（按频率分层）

- **ADC ISR（最高频）**
  - 采样 -> PLL -> 前馈 -> 电流参考 -> 电流环 -> PWM 刷新
  - 决定电流波形跟踪质量与动态性能

- **1kHz ISR（中频）**
  - 母线平均、电压环、RMS、参数自适应、软启动
  - 为快环提供目标值和动态参数

- **200Hz ISR（低频）**
  - 启动状态机、AC 输入保护、SCI 通信调度
  - 负责“上电到故障恢复”的运行管理

---

## 3. 三条必追变量链

### A. 控制链（闭环主线）
`ADC采样 -> 误差 -> PI -> 占空比 -> EPwm1Regs.CMPA`

### B. 相位链（同步主线）
`SADC.Vac -> PhaseLockLoop -> PLL.CntFianl -> sinetable -> Vac.Sine`

### C. 运行链（状态主线）
`Flag.PFCState -> Init/Wait/Rise/Run/Err -> Flag.PWM/Flag.Err/PFCok`

---

## 4. 读代码时只回答这 4 个问题

对任意函数，先写清：

1. 输入来自哪里（哪些全局量/寄存器）
2. 输出写到哪里（哪些全局量/寄存器）
3. 执行频率/时机（哪个中断里）
4. 对系统行为的直接影响是什么

> 一次读函数不要超过 2 层调用深度，优先打通主链路再回看细节。

---

## 5. 快速定位风险点

优先检查这些位置：

- 积分器限幅与输出限幅（防积分饱和）
- 状态切换条件与恢复条件
- 过压/欠压/过流触发与清除逻辑
- 多中断共享全局变量的更新时机

---

## 6. 建议产出（读完就能复盘）

建议你在阅读后沉淀三张小表：

1. **函数职责表**：函数名 / 频率 / 输入 / 输出
2. **变量字典表**：变量名 / 物理意义 / Q格式 / 更新源
3. **状态机表**：状态 / 进入条件 / 退出条件 / 故障动作

完成这三张表，基本就算“读懂项目”。
